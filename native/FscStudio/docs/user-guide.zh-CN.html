FSC Studio 上手指南

适用于原生 Windows 版。软件、推理模型和运行库均包含在安装包内，不需要安装 Python。

目录： 安装版本 首次使用 常用操作 页面说明 硬件加速 数据安全 故障排查

1. 选择安装版本

安装包适用设备推荐设置
FSC-Studio-Setup-x64.exe普通 Intel/AMD Windows 电脑，也适用于 NVIDIA 显卡电脑Auto，自动使用 DirectML GPU，失败时回退 CPU
FSC-Studio-CUDA-Setup-x64.exe具有兼容驱动的 NVIDIA 独立显卡电脑Auto，优先 CUDA；目标电脑不需要安装 CUDA Toolkit
FSC-Studio-Setup-arm64.exeSurface Pro 11 等 Snapdragon X 系列 Windows on ARM 设备Auto，按 NPU、Adreno GPU、CPU 顺序尝试

三种安装包安装的是同一个 FSC Studio，不需要同时安装。更换版本时先卸载旧版本；卸载不会删除你自行保存的 .fscdb 数据库。

本项目代码采用 Apache-2.0，但安装包内的 InsightFace buffalo_l 预训练权重不在该许可证范围内。上游将这些权重限定为非商业研究用途；商业使用需要另行向 InsightFace 取得许可。安装前请阅读第三方许可提示。

2. 首次使用

  1. 启动 FSC Studio，在左下角选择中文、English、日本語或한국어。
  2. 进入 Library，选择 Create Database 新建数据库，或选择 Open Database 打开已有的 .fscdb。
  3. 选择 Import Images 导入单张或多张图片，或选择 Import Folder 递归导入文件夹。
  4. 导入时预览框会显示正在分析的图片；检测、特征提取和数据库写入在后台执行。
  5. 进入 Runtime 查看实际使用的推理设备。显示的 provider 才是当前真正生效的设备。

3. 常用操作

用一张照片查找相似人脸

  1. 在 Search 打开查询照片。
  2. 照片有多张脸时，可点击预览框中的人脸框，或在人脸列表中上下切换。
  3. 设置相似度、结果数量、最低质量和身份严格度，然后开始搜索。
  4. 预览会节流显示正在比对的数据库图片，并在结束后停在最高相似度结果；该动画不会逐张阻塞搜索。

学习同一个人的多张照片

  1. 在 People 新建人物，并把确认属于该人物的人脸分配给他/她。
  2. 尽量提供 5 至 10 张清晰、角度和表情不同的照片。
  3. 点击 Train Identity Profiles。软件会生成 canonical exemplars、质量权重、hard negatives 和个体阈值。
  4. 样本少于 3 张的档案会标记为 weak，只给出复核建议，不自动高置信确认。

两张照片单独比对

在 Compare 分别打开图片 A 和 B。两边都会自动重新检测；有多张脸时可用列表或鼠标选择任一人脸。结果显示所选两张脸的 embedding 余弦相似度、质量和关键点信息。

摄像头实时识别

在 Camera 选择摄像头、数据库、识别间隔和身份模式后启动。画面采集与识别任务分离，短时投票只稳定姓名显示，不会把单帧的 unknown/review 强行改成 confirmed。

4. 页面说明

页面用途
Overview查看人脸、人物、标签、待复核和质量概况，并快速进入主要工作流。
Library创建/打开数据库、导入图片、筛选与批量编辑、导出 CSV；查看图像、3D Landmarks 和 Dense Mesh。
People维护人物、成员和备注，合并人物，查看身份模板健康度并训练 Identity Gallery。
Search单图多脸搜索，同时显示相似脸排名、人物候选、置信度和证据照片。
Camera摄像头实时检测、数据库搜索和稳定人物识别。
Review处理待复核队列，接受或拒绝 AI Suggested Person，并更新人物模板。
Clusters把尚未命名但可能属于同一人物的人脸聚类，并批量分配。
Compare独立比较两张图片中任意选中的两张脸。
Runtime切换/检查运行设备，执行完整性检查、备份、checkpoint、VACUUM 和受限旧版 DTB 转换。

各预览框保持原图比例，长边适配可用区域。选择人脸后可使用左上角的小型 Focus on Face 控件放大人脸，再切换回完整图像。Dense Mesh 支持 Points/Textured、拖动旋转、滚轮缩放、重置和可选 3D Landmarks 叠加。

5. 身份模式与硬件加速

身份严格度

  • Strict：默认且最安全，宁可进入 unknown/review，也避免高置信误认。
  • Balanced：在召回率与误认风险之间折中。
  • Broad：适合寻找候选，不建议把结果直接当作身份确认。

Surface Pro 11

  • QNN NPU：使用量化 InsightFace 模型运行检测、对齐关键点和 embedding 等适合 HTP 的固定推理任务。
  • QNN GPU：使用 Adreno 处理浮点模型；少量不支持的形状算子允许回退 ARM CPU。
  • CPU：兼容回退和数据库、聚类、模板评分等本来就更适合通用处理器的任务。
  • Dense Mesh 的浮点展示模型使用 Adreno GPU 或 CPU，不强行放到 NPU。

6. 数据安全

  • .fscdb 中包含照片路径、特征向量和身份模板，应按生物特征数据保护。
  • 重要操作前在 Runtime 创建备份，并定期运行 Integrity Check。
  • 导入或搜索都在本机完成，不需要上传云端。
  • 不要把私人数据库、照片或导出的 CSV 提交到公开 Git 仓库。

7. 常见问题

选择 Auto 后没有使用预期设备

在 Runtime 查看实际 provider 和回退原因。CUDA 版需要兼容的 NVIDIA 驱动；ARM64 QNN 版需要 Snapdragon X 系列设备及当前 Windows/Qualcomm 驱动。

识别人名不稳定或经常 unknown

先确认人物至少有 5 张多样、清晰、未忽略且已正确分配的照片，然后重新训练身份模板。不要通过降低阈值掩盖错误样本。

某张图片没有检测到脸

检查脸部是否过小、严重模糊或被遮挡。可以使用更高分辨率原图；程序不会用低可信检测强行写入数据库。

安装时出现 SmartScreen 提示

测试发行版尚未使用商业 Authenticode 证书。只从可信来源取得安装包，并核对发布者提供的 SHA-256；正式公开发行应签名。

FSC Studio 处理的是相似度和统计身份判断，不能代替法律、门禁或金融场景中的专业身份核验。
